Git Basics: Starting with the Mercury Github

Initial Setup

Clone the repository to your local machine:

git clone https://github.com/SoftwareBozos/MercuryRobotics.git

1. Grab Latest Github Changes

Check for changes:

git pull origin main

This replaces all changed files on your system with the new ones from Github

2. Make Changes

Edit files in your local repository

3. Double Check Changes

Check for changes:

git status

This stages all changes. To stage specific files, replace '.' with file names.

4. Stage Changes

Stage your changes:

git add .

This stages all changes. To stage specific files, replace '.' with file names. Also you can use '-A' instead of '.' to add all changed files.

5. Commit Changes

Commit your staged changes:

git commit -m "Your commit message here"

6. Push Changes

Push your commits to GitHub:

git push origin main

Replace 'main' with your branch name if the branch you are on is different.

Additional Git Commands

  • git pull: Update your local repository with changes from GitHub
  • git branch: List, create, or delete branches
  • git checkout: Switch branches or restore files
